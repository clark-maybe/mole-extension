## 5.1 从 Claude Code 学到的工程实践

### 模块化与单一职责

Claude Code 的代码库虽然庞大，但每个模块的职责都很清晰：

```typescript
// 好的例子：每个文件只做一件事
src/
├── QueryEngine.ts     // 只负责 AI 查询逻辑
├── cost-tracker.ts   // 只负责成本追踪
├── context.ts        // 只负责上下文收集
```

**启示**：即使项目变大，也要保持模块的清晰边界。当一个文件超过几千行时，可能需要考虑拆分了。

### 依赖注入与可测试性

Claude Code 大量使用了依赖注入：

```typescript
// QueryEngine 的配置中大量使用函数注入
export type QueryEngineConfig = {
  canUseTool: CanUseToolFn        // 函数而非直接依赖
  getAppState: () => AppState      // 函数而非直接依赖
  setAppState: (f: (prev: AppState) => AppState) => void
}
```

**启示**：通过依赖注入，代码更容易测试。你可以在测试中注入 mock 函数，而不需要启动整个系统。

### 错误处理的分层

Claude Code 对错误进行了精细的分层处理：

```typescript
// 错误分类
const categorizeRetryableAPIError = (error: Error) => {
  if (error instanceof RateLimitError) return 'rate_limit'
  if (error instanceof NetworkError) return 'network'
  if (error instanceof ServerError) return 'server'
  if (error instanceof AuthenticationError) return 'auth'
  return 'unknown'
}
```

**启示**：不要笼统地 catch 所有错误。分类处理让你可以针对不同错误采取不同的策略。

### 配置外部化

Claude Code 大量使用配置文件和环境变量：


**启示**：将可控的行为外部化为配置，可以让同一套代码适应不同场景，减少分支维护。

### 错误恢复策略

```typescript
// 实现重试机制
async function withRetry<T>(
  operation: () => Promise<T>,
  options: {
    maxAttempts: number
    backoffMs: number
    shouldRetry?: (error) => boolean
  }
): Promise<T> {
  let lastError

  for (let attempt = 1; attempt <= options.maxAttempts; attempt++) {
    try {
      return await operation()
    } catch (error) {
      lastError = error

      if (options.shouldRetry && !options.shouldRetry(error)) {
        throw error
      }

      if (attempt < options.maxAttempts) {
        await sleep(options.backoffMs * attempt)
      }
    }
  }

  throw lastError
}
```

## 5.3 总结与展望

### Claude Code 教给我们的最重要的事

#### 1. AI 是工具，不是魔法

Claude Code 的成功不是因为它用了什么神奇的 AI，而是因为它把 AI 用得恰到好处。工具系统、权限控制、错误处理——这些都是传统的软件工程实践。

**关键认识**：构建好的 AI 应用，80% 的工作是工程，20% 是 AI 调用。

#### 2. 安全永远是第一位的

Claude Code 的权限系统和沙箱机制提醒我们：给 AI 工具时，要像给实习生钥匙一样谨慎。

**最佳实践**：
- 默认禁止，必要时才开放
- 危险操作需要显式确认
- 敏感操作要有多层防护

#### 3. 用户体验藏在细节里

从"先读后写"的原则到 Git 提交的 HEREDOC 格式，Claude Code 的用户体验来自无数细节的积累。

**启示**：好的 UX 不是大改版，而是无数小优化的叠加。

#### 4. 成本意识不可或缺

Token 缓存、分层提示词、启动优化——Claude Code 在每一个环节都考虑了成本。

**实践建议**：
- 监控系统成本
- 优化提示词长度
- 实现智能缓存

### AI 开发的方向

Claude Code 的架构预示了 AI 应用开发的一些趋势：

| 方向 | 描述 |
|------|------|
| **代理化** | AI 从被动响应走向主动执行 |
| **多代理协作** | 复杂任务分解给多个专门化代理 |
| **工具化** | AI 通过工具与外界交互 |
| **安全优先** | 权限和沙箱成为标配 |
| **成本感知** | 开发者需要关注 AI 调用的成本 |

### 最后的思考

Claude Code 的源码是一个宝库，展示了如何将 AI 能力真正产品化、工程化。它不仅仅是 Anthropic 的产品，更是整个 AI 行业的一份珍贵教材。

希望这个系列文章能够帮助你更好地理解 AI 应用的构建之道。祝你在 AI 开发的道路上一路顺风！
